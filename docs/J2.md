# NewAPP GLPI - Documentation d'avancement

## Jour 2 - Module Kanban

Version : 1.0

Date : Juin 2026

---

# 1. Objectif

L'objectif est d'ajouter à NewAPP un système de gestion des tickets sous forme de Kanban.

Le projet est composé de deux parties :

* Backoffice : configuration du Kanban.
* FrontOffice : utilisation du Kanban.

---

# 2. Partie Backoffice

## 2.1 Fonction réalisée

Une nouvelle page de configuration Kanban a été créée.

Route :

```
/admin/kanban-config
```

Cette page permet de personnaliser les colonnes du Kanban.

---

## 2.2 Configuration disponible

Trois colonnes sont gérées :

| Statut      | Nom malgache | Statut GLPI |
| ----------- | ------------ | ----------- |
| Nouveau     | vaovao       | 1           |
| In progress | efa manao    | 2           |
| Terminé     | vita         | 5           |

---

## 2.3 Personnalisation

Chaque colonne possède :

* code technique ;
* nom français ;
* nom malgache ;
* couleur ;
* ordre d'affichage ;
* statut GLPI associé.

---

## 2.4 Stockage

Le stockage est effectué avec :

```
localStorage
```

Clé utilisée :

```
configuration_kanban_newapp
```

Fonctions disponibles :

```
obtenirConfigurationKanbanParDefaut()

chargerConfigurationKanban()

sauvegarderConfigurationKanban()

reinitialiserConfigurationKanban()
```

---

## 2.5 Interface Backoffice

La page affiche :

✓ Nom français

✓ Nom malgache

✓ Couleur

✓ Aperçu visuel

✓ Bouton Enregistrer

✓ Bouton Réinitialiser

✓ Message succès

✓ Message erreur

---

## 2.6 Intégration

Ajouts effectués :


### Layout

Utilisation du LayoutBackoffice existant.

# 3. Partie FrontOffice

## 3.1 Fonction réalisée

Une nouvelle page Kanban a été créée.

Route :

```
/front/kanban
```

---

## 3.2 Chargement

Au chargement :

* récupération configuration Kanban ;
* récupération tickets GLPI ;
* regroupement des tickets.

---

## 3.3 Colonnes

Trois colonnes sont affichées :

## Nouveau

Statut GLPI :

```
1
```

## In progress

Statut GLPI :

```
2
```

## Terminé

Statut GLPI :

```
5
```

---

## 3.4 Cartes tickets

Chaque ticket affiche :

* titre ;
* identifiant ;
* priorité ;
* date.

Chaque carte est cliquable.

---

## 3.5 Création rapide

Bouton :

```
Ajouter 1 ticket
```

Fonction :

* demande un titre ;
* crée le ticket dans GLPI ;
* recharge le Kanban.

---

## 3.6 Drag and Drop

Fonction disponible :

* glisser un ticket ;
* déposer dans une autre colonne ;
* mise à jour du statut GLPI ;
* rechargement automatique.

---

## 3.7 Confirmation

Lors du passage vers :

```
Terminé
```

Une confirmation est demandée.

Si annulée :

Le ticket reste dans sa colonne précédente.

---

## 3.8 Détail ticket

Au clic :

Une fenêtre affiche :

* ID ;
* titre ;
* description ;
* statut ;
* priorité ;
* date.

Bouton :

```
Fermer
```
